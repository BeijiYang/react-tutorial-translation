为什么不可变性很重要

为什么不可变性很重要

在之前的示例代码中，我们建议使用.slice()运算符来复制一份squares数组，再进行数据改动，以防将原有的数组修改。现在，我们来谈谈，这具体是什么意思，以及为什么重要。

通常来说，有两种方法来修改数据。第一种方法是通过直接改动变量的值来修改（mutate）数据，第二种是使用一份改动的拷贝后的数据，以此替换（replace）原有数据。

改动原数据

code

var player = {score: 1, name: 'Jeff'};
player.score = 2;
// 现在 player 是 {score: 2, name: 'Jeff'}
不改动原数据

code

var player = {score: 1, name: 'Jeff'};

var newPlayer = Object.assign({}, player, {score: 2});
// 现在 player 没有改变, 而 newPlayer 是 {score: 2, name: 'Jeff'}

// 或者使用对象展开符：
// var newPlayer = {...player, score: 2};

最终的结果是一样的，但是不直接改动基础数据的方法却能带来一些额外的好处：它有助于提升组件或者整个应用的性能。

更简单的 撤销/重做 和 穿越功能

不可改变性也让一些复杂的特性实现起来更容易。例如，在本教程后期，我们将要实现在棋局的不同阶段间穿越的功能。避免数据的变动，能让我们保存对旧版本数据的引用，当我们需要的时候，还能在它们之间切换。

追踪变动

对于被直接改动的对象，判断它们是否被修改是困难的。因为所以改动都直接在对象上进行。这要求对当前对象和之前的拷贝进行比较，追踪遍历整个对象数，比较每个变量和值。这个过程可能会非常复杂。

而判断不可变对象是否改变是非常容易的。 如果被引用的对象与以前不同，则对象已更改。就这么简单。

React中确定合适重新渲染

建立纯组件的时候，React的不可改变性带来的好处最明显。不可改变的数据能让我们很容易地确定改动是否发生，借此，可以确定组件何时要求被重新渲染。
